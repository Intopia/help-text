The problem

Help text is information associated with an individual form control that provides information for the user on how to fill in the field correctly. If the help text is not correctly associated with the relevant form controls, assistive technology users may not hear these text labels announced as they navigate through forms.

Help text should be programmatically associated with its relevant form controls.

Test criteria

  1. Is the help text announced when the form control recieves focus?

Test cases

Example 1: No association

Help text 
<div class="input">
  <label class="input__label" for="example01">Example 1</label>
  <span class="message-hint">Help text</span>
  <input class="input__control" type="text" id="example01">
</div>

Example 2: Label wrapped around help text

<div class="input">
  <label for="example02">
    <span class="input__label">Example 2</span>
    <span class="message-hint">Help text</span>
    <input class="input__control" type="text" id="example02">
  </label>
</div>

Example 3: Using the aria-describedby and id attributes

Help text 
<div class="input">
  <label class="input__label" for="example03">Example 3</label>
  <span class="message-hint" id="message03">Help text</span>
  <input class="input__control" type="text" id="example03" aria-describedby="message03">
</div>

Recommendations

While Example 2: Label wrapped around help text is well supported across Assistive Technologies, Example 3: Using the aria-describedby and id attributes is the recommended solution as it is often easier to apply within complex systems.
